Accessing ARexx port

AmiBroker has a built-in ARexx port with a powerful command set that enables you to write custom scripts for performing different tasks. A sample ARexx script is included.

AmiBroker ARexx port is accessible by the name "AMIBROKER.x", where x is 1, 2, 3, ...

Actual port name can be found in the ‘About’ requester, but in most cases you can assume that the port name is "AMIBROKER.1".

Writing custom ARexx scripts

When using any of the AmiBroker’s internal commands (described below), please keep in mind that ARexx commands are parsed twice: first by the ARexx server while interpreting the script, second by AmiBroker using the

dos.library/ReadArgs(). Then, please, put the commands into single quotes and double-quote those parameters that include spaces to prevent them from being regarded as string delimiters.

Example:

‘REQUEST BODY="This is an example" TITLE="Title with spaces" BUTTONS="Yes|Oh no" ‘

AmiBroker passes command results to your script using a special ARexx variable, RESULT. The error code is passed in the RC variable. Possible values for the error code are:

0 - success
5 - warning
10 - error
20 - fatal error

 

Commands Reference

ADDQUOTATION

ADDQUOTATION STOCK/A,DATE/A,PRICE=CLOSE/A,OPEN/K,HIGH/K,LOW/K,VOLUME/K

Adds a quotation to the database. If the selected quotation already exists, this function modifies existing price/volume data.

STOCK - (STRING) specifies the stock name (case sensitive)
DATE - (STRING) YY-MM-DD, quotation date
CLOSE - (FLOAT) gets the closing price
PRICE - synonym for CLOSE
OPEN - (FLOAT) gets the opening price
HIGH - (FLOAT) gets the day’s high
LOW - (FLOAT) gets the day’s low
VOLUME - (INT) gets the volume of trade

This function will fail if there’s no stock with the specified name - use ADDSTOCK to add a new stock name to the database. This function does NOT refresh the charts. Use REFRESH or ARRANGE to do that.
The parameter template for this command has changed in 1.7. Pre-1.7 scripts using keywords will work OK, but others will not.

ADDSTOCK

ADDSTOCK NAME=STOCK/A, CONT/S

Adds a new stock to the database

STOCK - (STRING) defines the stock name (case sensitive)
NAME - synonym for STOCK
CONT - switches continuous trading mode on.

ARRANGE

ARRANGE

Arranges and refreshes all open chart windows

ASK

ASK STOCK/K,DATE=QUOTATION/K

Asks whether the given stock/quotation exists in the database

STOCK - stock name to search for
DATE - date of quotation to search for

You can check only if a given stock exists using this command without the DATE parameter. Returns 0 if the given stock/quotation doesn’t exist in the database or 1 if it exists.

DATABASE

DATABASE CHANGE/S,NOCHANGE/S,SAVE/S,STATUS/S, OPEN/K

Database control

CHANGE - switches database status to changed
NOCHANGE - switches database status to not changed
SAVE - saves database to disk (if changed)
STATUS - queries database status, returns:
1 - if the database has been changed
0 - if the database has not been changed
OPEN - (NEW) OPEN=work:data/nyse - path to the database

Note: To force saving, use: ‘DATABASE CHANGE SAVE’

After the OPEN keyword, you should specify the path to the database directory (without trailing slash).
With _single_ DATABASE command you can now force saving the previous database and opening a new one. Example:
DATABASE CHANGE SAVE OPEN=your_path

DEBUG

DEBUG ON/S,OFF/S

Switches ARexx debug messages.

If ON, error/warning messages will be displayed in the appropriate requester.

GETFINANCE

GETFINANCE NAME=STOCK/A,QUARTER/N/A,YEAR/S,INCOME/S,EBT/S,EAT/S

This command retrieves financial data.

NAME=STOCK - short stock name (ticker) for which information is requested
QUARTER - quarter number (1-4)
Optional parameters:
YEAR - gets the year number
INCOME - gets the sales income
EBT - gets the earnings before tax
EAT - gets the earnings after tax

Command output is stored in the ARexx result string.

GETQUOTATION

GETQUOTATION STOCK/A,DATE/A,PRICE=CLOSE/S,OPEN/S,HIGH/S,LOW/S,VOLUME/S,INFO/S

Gets quotation data.

STOCK - (STRING) specifies the stock name (case sensitive);
DATE - (STRING) YY-MM-DD, quotation date;
CLOSE - gets the closing price;
PRICE - synonym for CLOSE;
OPEN - gets the opening price;
HIGH - gets the day’s high;
LOW - gets the day’s low;
VOLUME - gets the volume of trade
INFO - gets WSE-specific info (version 2.5 and up)

Result: A string containing selected data in this order:
<close> <open> <high> <low> <volume>
with a single space between actual numbers

Examples:

GETQUOTATION STOCK=APPLE DATE=96-01-03 CLOSE

This will return only the closing price.

GETQUOTATION STOCK=APPLE DATE=96-01-03 CLOSE VOLUME

This will return the closing price and volume number.

GETSELECTED

GETSELECTED STOCK/S,DATE/S

Gets the selected stock name or quotation date.

STOCK - gets the selected stock name
DATE - gets the selected quotation date

GETSTOCKINFO

GETSTOCKINFO NAME=STOCK/A, FULL/S, CODE/S, ADDRESS/S, ISSUE/S, NOMINAL/S, BOOKVALUE=BV/S, MARKET/S, GROUP/S, CONT/S, FAVOURITE/S, INDEX/S

This command retrieves stock information:

NAME=STOCK - short stock name (ticker) for which information is requested
Optional parameters:
FULL - gets the company's full name
CODE - gets the Polish stock code (INT)
ADDRESS - gets the company's address
ISSUE - gets the stock issue (INT)
NOMINAL - gets the nominal value (FLOAT)
BOOKVALUE - gets the book value (in thousands)
MARKET - gets the market ID
GROUP - gets the group ID
CONT - continuous quotation flag ( 0 = price fixing, 1 - continuous quotation enabled)
FAVOURITE - gets the favourite flag
INDEX - gets the index flag

Command output is stored in the ARexx result string. This command can retrieve one or more of the listed items. When retrieving multiple items, the result string will contain all selected item values, concatenated with a single space between them. Keep in mind that FULL and ADDRESS data can contain strings with spaces, so it will be difficult to separate them from other data! If you want to retrieve FULL or ADDRESS data, please use the following format:

GETSTOCKINFO <your_stock_name> FULL
or
GETSTOCKINFO <your_stock_name> ADDRESS

In this way you will get the information you need in the result string, with no further parsing necessary.

GUI

GUI LOCK/S,UNLOCK/S

Function: Locks/unlocks user interface

You can nest several GUI LOCK commands, but you have to call GUI UNLOCK exactly the same number of times to actually unlock the user interface.

Please use it with extreme care! If you fail to unlock it properly, the GUI user will not be able to exit the program!

IMPORT

IMPORT ASCII/S,TELETEXT/S,KING/S,FILE/K

Imports quotation data from a specified file.

ASCII - selects ASCII import mode
TELETEXT - selects TELETEXT mode (Warsaw Stock Exchange)
KING - selects KING mode (WSE)
FILE - (STRING) specifies file name. If none is specified, brings up a file requester.

QUIT

QUIT FORCE/S

Quits the program.

FORCE/S - suppresses the requester asking for confirmation.

REFRESH

REFRESH

Redraws all open charts

REMQUOTATION

REMQUOTATION STOCK/A,DATE/A

Removes the specified quotation of a stock.

STOCK - (STRING) the stock name (case sensitive)
DATE - (STRING) the quotation date (yy-mm-dd)

This function does NOT refresh the charts. Use REFRESH or ARRANGE to do that.

REMSTOCK

REMSTOCK NAME=STOCK/A

Removes the specified stock from the list.

STOCK - (STRING) the stock name (case sensitive)
NAME - the same as STOCK (a synonym)

This function can NOT be undone. It removes all stock data from memory AND from the disk! Make sure you know what you are doing!

REQUEST

REQUEST BODY/K,TITLE/K,BUTTONS/K,FILE/S,PATH/K,PATTERN/K,SAVE/S,STRING/S

Shows the ‘easy’ requester.

BODY/K - (STRING) defines the requester body text. Multiple lines are allowed; use ‘|’ as a new-line symbol.
TITLE/K - (STRING) the requester title
BUTTONS/K - (STRING) the requester buttons. Multiple buttons are allowed; use ‘|’ to separate individual button texts.

New parameters (available since 2.6) are:

FILE - displays a file-requester (BOOL)
PATH - specifies the initial path for the file-requester (STRING)
PATTERN - specifies the file matching pattern (STRING)
SAVE - puts the file-requester into save mode
STRING - displays a string-requester (BOOL)

NOTE:

The BODY parameter is now defined with /K (it was /A in 2.5 and earlier versions). So, you have to specify the BODY keyword, since this parameter is now optional (and not used in the file-requester). Thanks to the extended syntax of the REQUEST command, it now handles 3 types of requesters:

  • 'easy' requester - a simple message box with multiple answer buttons.
    Example:
    REQUEST BODY="Do you want to continue?" TITLE="Please confirm" BUTTONS="Yes|No"
    Returns: the number of the selected button, in order 0, 1, 2, ... (from left to right)

  • File requester - asks for a file name.
    Example:
    REQUEST FILE PATH=RAM: PATTERN=#?.h SAVE TITLE="Enter file name to save" BUTTONS=Save
    Returns: the selected file name

  • String requester - asks for a string.
    Example:
    REQUEST STRING BODY="Enter some text" TITLE="Just an example" BUTTONS="Ok|No"
    Returns: the entered string

SCREEN

SCREEN FRONT/S,BACK/S

Brings the AmiBroker screen to FRONT or puts it to BACK

SELECT

SELECT STOCK/K,DATE/K

Selects a stock and/or quotation (and marks it with a hairline on the chart).

STOCK - (STRING) the stock name (case sensitive)
DATE - (STRING) the quotation date (yy-mm-dd) if the specified one doesn’t exist, the nearest next will be selected.

From version 2.5 you can also use special names @NEXT and @PREV to go to the next/previous stock in the list.

Example: SELECT STOCK=APPLE /* selects 'APPLE' */
SELECT STOCK=@NEXT /* selects the next stock in the stock list */

From version 2.6 SELECT can switch to the first/last/next/previous stock or quotation thanks to the new special codes @FIRST, @LAST, @NEXT, @PREV that can be used in place of the short stock name (ticker) or quotation date.

Examples:

SELECT STOCK=@FIRST
SELECT STOCK=@LAST
SELECT STOCK=@NEXT

SELECT DATE=@LAST
SELECT DATE=@PREV
SELECT DATE=@FIRST

NOTE:

SELECT DATE=@FIRST will select the first VISIBLE quotation on the chart. To make sure that the truly first quotation is selected, use the following sequence:
ZOOM ALL
SELECT DATE=@FIRST

SETFINANCE

SETFINANCE NAME=STOCK/A,QUARTER/N/A,YEAR/N,INCOME/N,EBT/N,EAT/N

Function: Sets financial data.

NAME=STOCK is the ticker
QUARTER - is the quarter number (1-4)
YEAR - is the year to set
INCOME - is the sales income for the selected quarter (in thousands)
EBT - are earnings before taxes for the selected quarter (in thousands)
EAT - are earnings after taxes for the selected quarter (in thousands)

SETSTOCKINFO

SETSTOCKINFO NAME=STOCK/A, FULL/K, CODE/N, ADDRESS/K, ISSUE/N, NOMINAL/K, BOOKVALUE=BV/N, MARKET/N, GROUP/N, CONT/S, FIXED=NOCONT/S, FAVOURITE/S, NOFAVOURITE/S, INDEX/S, NOINDEX/S

Function: Sets stock information.

NAME=STOCK is the stock short name (ticker)
FULL - is the full name of the stock
CODE - is the 4-digit code (Polish)
ADDRESS - is the company's address
ISSUE - is the number of shares issued
NOMINAL - is the nominal stock value
BOOKVALUE - is the book value of the company (in thousands $)
MARKET - is the zero-based market identifier (number)
GROUP - is the zero-based group identifier
FIXED - switches on price fixing (and line charts)
CONT - switches on continuous quotations (and candlestick charts)
FAVOURITE - switches on the favourite flag
NOFAVOURITE - switches off the favourite flag.
INDEX - switches on the index flag
NOINDEX - switches off the index flag.

VERSION

VERSION

Returns the program version tag (STRING).

ZOOM

ZOOM SIZE/N,ALL/S

Function: Zooms the chart.

SIZE - specifies the number of quotations to show in the chart
ALL - switches the chart to show all available quotations
