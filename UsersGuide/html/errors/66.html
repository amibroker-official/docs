Error 66. Variables with 'name' prefix are inaccesible via StaticVar* in this formula. This prefix is reserved by #pragma enable_static_decl

You attempted to use StaticVarGet or StaticVarSet or StaticVarAdd on a static variable that was declared via #pragma enable_static_decl

If you use #pragma enable_static_decl with certain prefix, these variables are only accessible via identifier and not via old style StaticVarGet/StaticVarSet to avoid problems. Declared static variables are read once (when they are declared) and updated once (when formula finishes). Using StaticVarGet/Set for such static variables would interfere with read once/write once methodology so such attempts are detected and prevented (by displaying error 66).

 

// the pragma below is required to enable support for static declarations
// it also defines the prefix which is added to variables declared with static keyword
// it is one prefix per formula.
#pragma enable_static_decl(prefix)

// it is really good idea to consequently use naming convention that
// distinguishes declared static variables from the others
// we highly recommend using UNDERSCORE '_' before all declared static variables
// so they can be easily spotted in the code
// The other choice could be s_ prefix

static _myvar;  // note: you can NOT access such variable via StaticVarGet/Set
printf( "This variable is really static %g", _myvar++ ); // you CAN access it directly (as any other variable)


// keep in mind that for speed static variable is read only once at the declaration
// and saved only once - when formula execution is completed
// This way declared static variables offer same speed as regular variables (no speed penalty)

StaticVarGet("prefix_myvar"); // attempt to use StaticVarGet on declared variable would trigger error 66

StaticVarGet("other_variable_is_ok"); // accessing OTHER (non-declared) variable is OK
